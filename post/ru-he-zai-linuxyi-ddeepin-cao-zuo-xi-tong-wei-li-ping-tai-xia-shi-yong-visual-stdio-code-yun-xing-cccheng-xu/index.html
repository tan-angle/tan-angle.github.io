如何在Linux（以Ddeepin操作系统为例）平台下使用visual stdio code运行C/C++程序

2021-06-04 / 6 min read

一、前期准备

  1. 首先，你需要在visual stdio code官网或者deepin应用商店下载并安装好visual stdio code。

  2. 其次，你需要在Linux终端中安装gcc、g++和gdb。因为vscode只是一个集成开发工具（IDE），并没有自带有C/C++的编译器，因此我们需要去安装一个C/C++编译器。它不需要像在Windows操作系统下先安装MinGw-w64，然后找到MinGw-w64的Bin目录，然最后配置环境变量。

  3. 打开终端依次输入下面的命令，确保安装好gcc、g++和gdb。这里的四个过程我就不一一截图啦！根据系统提示操作即可，不会有啥大问题。

①更新软件包

sudo apt-get update
[根据系统要求如果需要输入密码则输入密码]
[如果在安装过程中有提示需要输入Y(Yes)/N(No)，没有什么特殊要求一般输入Y]

②安装gcc

sudo apt-get install gcc
[根据系统要求如果需要输入密码则输入密码]
[如果在安装过程中有提示需要输入Y(Yes)/N(No)，没有什么特殊要求一般输入Y]

③安装g++

sudo apt-get install g++
[根据系统要求如果需要输入密码则输入密码]
[如果在安装过程中有提示需要输入Y(Yes)/N(No)，没有什么特殊要求一般输入Y]

④安装gdb

sudo apt-get install gdb
[根据系统要求如果需要输入密码则输入密码]
[如果在安装过程中有提示需要输入Y(Yes)/N(No)，没有什么特殊要求一般输入Y]

二、打开visual stdio code，安装相关插件


安装好visual stdio code后，需要安装一下三个插件：

  1. Chinese (Simplified) Language Pack for Visual Studio Code（中文语言包），因为刚开始第一次安装好visual stdio code默认显示的是英文，所以需要它设置为咱们Chinese people看得懂的字！！！
  2. C/C++
  3. Code Runner
    这里就不过多赘述，如果这一步看不懂的可以看看我之前写的一篇“浅谈一下如何在Windows平台下使用VS Code（全称visual studio code）编写、运行和调试C/C++程序”文章，里面已经讲得非常详细啦！！！

三、编译、运行程序

首先，你需要新建一个C/C++的项目，其次新建好一个C/C++文件，最后在C/C++文件的代码编辑区编写一个超级简单的程序（Hello World！），实在是懒得敲的，我也给你准备好啦。这一步也不会的同样看我这一篇文章。

#include <stdio.h>
int main(){
   int a=1,b=2;
   printf("a+b= %d",a+b);
   return 0;
}

写完程序记得保存哦！

1.点击代码编辑区右上角的三角形按钮(原本这个位置没有，是安装Code Runner插件后新增的一键编译运行程序的按钮)，或者按Ctrl+Alt+N就能编译运行当前程序。

2.在终端你应该会看到这样的结果。

四、调试程序

首先，你应该也必须养成的一个习惯就是在电脑上桌面或者其他地方的位置新建一个项目（文件夹），用于存放项目编写的代码。
注意：配置好的调试环境仅会对存放在这一个文件夹以及这个文件夹子目录里的程序生效！
① 在VS code里面点击：文件>打开文件夹，打开刚刚新建的项目（文件夹）

② 再在刚才的项目下新建一个 .test文件夹(注意前面的"."号)
然后在.test文件夹下新建两个json文件分别叫做:launch.json和 tasks.json，然后分别复制下面这两个文件到相应的位置。

  • launch.json
{
    "version": "0.2.0",
    "configurations": [
        {
            "name": "C/C++",
            "type": "cppdbg",
            "request": "launch",
            "program": "${fileDirname}/${fileBasenameNoExtension}",
            "args": [],
            "stopAtEntry": false,
            "cwd": "${workspaceFolder}",
            "environment": [],
            "externalConsole": false,
            "MIMode": "gdb",
            "preLaunchTask": "compile",
            "setupCommands": [
                {
                    "description": "Enable pretty-printing for gdb",
                    "text": "-enable-pretty-printing",
                    "ignoreFailures": true
                }
            ]
        }
    ]
}
  • tasks.json
    如果是c语言就需要将下面的command项由g++改为gcc
{
    "version": "2.0.0",
    "tasks": [{
            "label": "compile",
            "command": "g++",
            "args": [
                "-g",
                "${file}",
                "-o",
                "${fileDirname}/${fileBasenameNoExtension}"
            ],
            "problemMatcher": {
                "owner": "cpp",
                "fileLocation": [
                    "relative",
                    "${workspaceRoot}"
                ],
                "pattern": {
                    "regexp": "^(.*):(\\d+):(\\d+):\\s+(warning|error):\\s+(.*)$",
                    "file": 1,
                    "line": 2,
                    "column": 3,
                    "severity": 4,
                    "message": 5
                }
            },
            "group": {
                "kind": "build",
                "isDefault": true
            }
        }
    ]
}

好了，到这里就已经可以正常调试啦！此时也可以按 Ctrl+Shift+B 直接调用配置好的g++ task 编译程序而不运行程序。
