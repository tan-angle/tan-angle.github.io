浅谈一下如何在Windows平台下使用VS Code编写、运行和调试C/C++程序

2021-06-04 / 8 min read

亲自查找资料解决的，各位小可爱看到此内容后并为你解决了麻烦或者困惑，恳请在看完此内容后给个关注或者点赞。同时也感谢发表与此内容有关的博主，我也不乏借鉴和学习他们所写的优秀博客。

友情提醒：本次内容是在Windows系统下完成的。如果是初学者，可以试着按照以下教程 or 方法直接操作试一试，期间有任何问题或者觉得我哪里讲得不详细的地方都可以底下评论 or 私信作者哟！

一、前期准备工作

1.你需要下载一个MinGW-w64并把它安装好。现在你就可以点击MinGW-w64这几个字，之后会自动跳转到下载的地方。

2.配置系统环境变量

（1）在桌面选择“我的电脑”，右键选择属性。

（2）选择高级系统设置。“我的电脑”的高级系统设置可能和大家的不太一样，因为我现在所使用的电脑版本是win10_20H2版本。

大家的应该是下面这个页面：

（3）选择环境变量

（4）在系统变量中点击Path之后，点击编辑

（5）找到你前面安装好的MinGW-w64文件夹，并进入到bin目录，之后复制此路径。在第四步点击“编辑”后，进入到Path变量里面，选择“新建”，之后添加MinGW-w64的bin目录路径，最后点击“确定”。离开此页面后还有两个页面需要点击“确定”（不出意外的话），正常点就行。

（6）到这里准备工作已经步入尾声啦！最后咱们需要进行测试一下看看自己配置的环境变量成功了嘛！按住win+R键输入cmd，点击确定。

（7）输入g++ -v或者gcc -v，出现如下界面内容，则恭喜你配置成功啦！没有出现此内容的小伙伴也不要灰心，重新按照上面的步骤再执行一次，肯定是没有问题的，加油哟！

二、搭建VS Code环境

  1. 首先你需要在微软官网或者Windows商店中下载并安装好VS Code，具体安装教程可自行BaiDu。
  2. 安装成功后，打开VS Code并点击里面的插件store，搜索并安装好以下几个插件：

①Chinese (Simplified) Language Pack for Visual Studio Code（中文语言插件）因为刚开始下载好并打开此应用的语言默认为English，所以需要用此插件汉化以下，转化为咱ZG人看得懂的字。

②C/C++（C/C++扩展在VisualStudio代码中添加了对C/C++的语言支持，包括IntelliSense和调试等功能）

③Code Runner（用于运行C, C++, Java等语言的插件）

二、C/C++代码编写

在完成第一步的环境搭建步骤后就看可以编写C/C++代码啦！那具体怎么编写呢？下面我就从项目（文件夹）的创建、打开、C/C++文件的创建和编写以及运行几个步骤尽我所能的和大家讲清楚！

1.项目的创建：在电脑桌面建一个文件夹（文件夹的名字自己随便，但是作为我国未来的高科技人才你怎么能随便命名呢？哈哈哈哈），我的建议就是按照项目的英文名字来命名，不断养成“看名知其意”的好习惯，千万不要给我养成“见异思迁”那种哈，不然小心我给你来一个垫炮加杵炮。我这里就以IT行业最著名也是最出名的Demo项目来演示。

2.项目的打开：选中Demo项目，右键鼠标点击通过Code打开，之后会自动进入到VS Code中。进入到VS Code之后，不出意外的话会看到DEMO名字的项目。

3.C/C++文件的创建和编写：点击第一张图片的地方，就会出现第二张图的文件，之后根据自己的喜欢命名文件即可！（文件后缀名写.c或者.cpp）。双击鼠标左键打开My_Demowen文件后就可以在代码编写区敲C/C++代码啦。

大家如果实在是不想自己动手敲代码，可以把下面这段代码复制进去：

#include<stdio.h>
int main()
{
    printf("这是我的第一个Demo项目！");
    return 0;
}

4.C/C++文件的运行：点击①处的地方就可以在②处看到运行的结果。

运行结果：发现结果是乱码的（也就是出现看不懂的字），但事实上与我们期望的结果不一致，那怎么办？好啦，今天就先讲到这里吧，下课！

5.代码运行的时VS Code有时候可能会出现“Code is already running”的代码提示（特别是在代码中有Scanf函数的时候），而且没有输出内容怎么办？重启VS Code还是不行，照样有错误提示。

解决办法：同时按住Ctrl+Alt+M。然后点击设置，选择用户，在搜索框里输入run code，找到Run In Terminal选项把它勾选上就可以啦，最后再回到代码编辑区点击运行就可以终端输入和看到输出结果咯！

三、程序调试

在你的项目文件夹（我的项目是Demo）下新建两个.json文件分别叫做

  • launch.json
  • tasks.json

然后将下面的代码复制到对应的.json文件中去并保存，要仔细阅每个图后面的说明性文字。

（1）launch.json

{
    "version": "0.2.0",
    "configurations": [
        {
            "name": "C/C++",
            "type": "cppdbg",
            "request": "launch",
            "program": "${fileDirname}/${fileBasenameNoExtension}.exe",
            "args": [],
            "stopAtEntry": false,
            "cwd": "${workspaceFolder}",
            "environment": [],
            "externalConsole": false,
            "MIMode": "gdb",
            "miDebuggerPath": "gdb.exe",
            "preLaunchTask": "compile",
            "setupCommands": [
                {
                    "description": "Enable pretty-printing for gdb",
                    "text": "-enable-pretty-printing",
                    "ignoreFailures": true
                }
            ]
        }
    ]
}

launch.json效果图：

（2）tasks.json

如果你所编写的代码是C语言将下面的command项由g++改为gcc。如果是多文件编译（即函数声明和函数定义分开，不懂的别乱改），需要将args列表中的"${file}"项修改为"${workspaceFolder}/*.cpp" ，多文件推荐用cmake。

{
    "version": "2.0.0",
    "tasks": [
        {
            "type": "shell",
            "label": "compile",
            "command": "g++",
            "args": [
                "-g",
                "${file}",
                "-o",
                "${fileDirname}\\${fileBasenameNoExtension}.exe"
            ],
            "problemMatcher": [
                "$gcc"
            ],
            "group": {
                "kind": "build",
                "isDefault": true
            }
        }
    ]
}

tasks.json效果图：

（3）配置完上面的json文件后，现在就可以进行调试啦，并且可以按 ctrl+shift+b 直接调用配置好的task编译而不运行程序，类似于一些IDE的编译选项。
